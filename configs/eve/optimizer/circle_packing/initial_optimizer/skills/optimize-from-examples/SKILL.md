Use this skill for either Phase 2 solver optimization or Phase 4 optimizer optimization.

1. Read `README.md` first to confirm which phase you are in and what the reference example directories, `guidance/`, and `output/` mean in this workspace.
2. Inspect the relevant files under the available reference example directories before editing anything. Treat them as reference material, not as files to modify.
3. Start from the existing `output/` contents.
4. Keep the result internally consistent with the current workspace phase:
   - In Phase 2, optimize the solver candidate in `output/`.
   - In Phase 4, optimize the optimization guidance files in `output/`.
